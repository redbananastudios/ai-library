# Content Workflows

Reserved for cross-channel content workflows — pipelines where a single concept produces outputs for multiple channels at once (e.g. one concept → social post + blog article + ad variants).

Empty in v1. The single-channel workflows live in `../social/`, `../blog/`, `../ads/`.

When cross-channel composition becomes a recurring need, workflows like `content/campaign_launch.yaml` or `content/product_drop.yaml` will land here.
