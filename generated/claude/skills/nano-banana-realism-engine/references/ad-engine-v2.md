# Nano Banana Ad Engine v2

Use this mode when the user wants a complete ad-creative package, not just prompts.

## Deliverable per concept

For each of 3 to 5 concepts, produce:

1. **Concept name**
2. **Audience moment**
3. **Hook**
4. **Headline**
5. **Primary image prompt**
6. **A/B variant image prompt**
7. **Why this works**

## Hook rules

- One sentence or fragment
- Short enough for paid social testing
- Focus on pain, aspiration, proof, ease, speed, status, or transformation
- Avoid generic hype language

## Headline rules

- 3 to 10 words
- Specific beats vague
- Can be product-led, benefit-led, or outcome-led

## Primary vs A/B image prompt rules

The A/B variant must test a real visual hypothesis, for example:
- same offer, different emotional tone
- same story, different camera preset
- same product, different use setting
- same audience, different level of polish

Do not produce near-duplicate prompts.

## Recommended concept spread across 3 to 5 outputs

Aim to distribute concepts across these buckets when appropriate:
- one candid UGC-style concept
- one cleaner commercial concept
- one problem-aware concept
- one aspiration-led concept
- one niche audience or local-context concept

## Useful agent behavior

If the user gives only a product or service:
- infer a likely audience
- infer a Facebook-suitable concept spread
- state assumptions briefly at the top

If the user gives a brand:
- read the project-local memory layer first when available
- keep hooks and visuals consistent with that brand profile

## Optional add-ons

If the user asks for a more complete package, also provide:
- caption starter lines
- suggested aspect ratio per concept
- creative testing notes
- a compact negative prompt shared across all concepts
