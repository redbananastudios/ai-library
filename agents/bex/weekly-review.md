**Posted by Bex to `#bex` on Monday at 09:00.**

## Format

```
**Last week — wins**
• [brand]: [outcome with number where possible]
• ...

**Last week — misses**
• [brand]: [what didn't happen and why]
• ...

**This week — priorities**
• Willow: [3 items]
• Marley: [3 items]
• First Taxis: [3 items]
• Red Taxi: [3 items]
• John Wood: [3 items]
• Red Banana: [3 items]
• FBA: [3 items]
• Me (Bex): [3 items]

**Workload**
• Viktor: [N] open tasks, [M] due this week — [status: ok / heavy / light]
• Me: [N] open tasks across brands — [status]

**Trends (analytics synthesis)**
• [brand] [metric]: [direction with number, e.g. "Marley GBP profile views +18% wk/wk"]
• ...

**Subscription renewals coming up (next 14 days)**
• [service] — £[amount] — renews [date] — [keep/review/cancel recommendation]
• ...

**Role gap signals** (if any)
• [Domain] work happened N times this week — would be [Sasha/Mira/etc.] if she existed. [Recommend build / monitor]
```

## Rules

- **"Wins" and "misses" must be specific.** "Marketing improved" is not a win. "Marley CTR went from 0.7% to 1.4%" is
- **"This week's priorities"** — 3 max per brand, picked by impact
- **"Workload"** — surface imbalance: if you (Bex) have >20 open tasks across brands, that's a flag. If Viktor has 0, ask Peter what to push to him
- **"Trends"** — only the 3-5 most important deltas
- **"Subscription renewals"** — surface anything in the next 14 days
- **"Role gap signals"** — this is your honest read of whether a framework-only specialist is starting to be needed. Don't manufacture gaps; only flag if work in that domain has actually happened ≥3 times in the week

## Reading order

1. ClickUp: tasks closed last week (shipped only) — pick top 3-5 wins
2. ClickUp: tasks that slipped (overdue without explanation) — pick misses
3. ClickUp: tasks open this week, by brand
4. ClickUp: count open tasks by owner (you / Viktor) — flag imbalance
5. Analytics data — top trends across brands (GA4, GBP, Shopify, etc.)
6. Subscription database — renewals within 14 days
7. Decisions log + framework-only role tracking — any domain hitting recurrence threshold

## After posting

- Peter responds with adjustments to "This week's priorities"
- For each subscription renewal flagged, Peter says keep / review / cancel
- For role gap signals, Peter decides: keep absorbing or start building that specialist
- You update ClickUp priorities and subscription database accordingly
