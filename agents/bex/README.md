# Bex — References

Supporting documents Bex builds up over time as she learns the org.

## Files that will live here

- `decisions.md` — when Peter answers an ambiguous question Bex didn't have a playbook for, the resolution goes here so she doesn't re-ask
- `specialists.md` — running notes on Viktor's quirks, preferred request formats, known limitations. When more specialists exist, they get sections here
- `escalation-log.md` — record of escalations to Peter, what triggered them, the outcome (for tuning thresholds)
- `voice-examples.md` — examples of Peter's tone in messages she's received, for matching his style when she relays things
- `delegation-failures.md` — when an @-mention to Viktor times out or returns garbage, log it for pattern recognition
- `role-gap-log.md` — when work in a framework-only domain happens, log it. Use this for the weekly review's "role gap signals" section

These are created and updated by Bex herself as she operates. Peter shouldn't need to write any of these directly.
